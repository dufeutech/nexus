Verify before trusting in production:
  1. Token issuer matches:   {{ .Values.oidc.issuer }}
     A token whose `iss` differs is rejected by Envoy jwt_authn (fail-closed).
  2. Authorization is nexus-authored (nexus-native-authorization): the OIDC provider
     is NOT an authorization source. Provision roles/entitlements via the authz-admin
     surface; a subject with no nexus grants is authenticated but unprivileged
     (deny-by-default). Bootstrap the first admin with authzAdmin.bootstrapAdminSub.
{{- if not .Values.authzAdmin.authDisabled }}
  3. The admin-token secret "{{ include "identity-plane.authzAdminSecretName" . }}" exists and is valid.
{{- end }}

Smoke test (from inside the cluster):
  TOKEN=<a real access token from your OIDC provider>
  curl -s -H "Authorization: Bearer $TOKEN" https://{{ .Values.edge.ingress.host }}/ | grep -i '^x-user'
  # expect x-user-id / x-user-roles / x-user-suspended (roles are nexus-authored;
  # x-user-org and x-user-roles-source are retired)
