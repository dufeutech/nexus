Order matters (RFC C17): an unknown/unverified host is rejected (404) BEFORE auth,
so even a valid token cannot reach a backend for an unregistered tenant; a known
host with a missing/invalid token is rejected (401) by jwt_authn.

Before trusting in production:
  1. Register routing config via the routing control plane (administrative boundary):
     kubectl -n {{ .Release.Namespace }} port-forward svc/{{ include "edge-platform.routingFullname" . }}-control-plane 9400:9400
     # then POST /tenants and verified /domains (see routing-plane NOTES)
  2. Confirm the ZITADEL issuer above matches what your ZITADEL mints (RFC §6.6).
  3. Provide the admin PAT secret for sync-worker/reconciler (identity-plane.zitadel.patSecret).
